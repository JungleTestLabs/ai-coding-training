## 二、预习清单

### 🔴 必修 1：LLM 核心概念速通（30 分钟）

培训中频繁出现的词，课前请自行搞懂：

| 概念 | 一句话解释 | 为什么重要 | 预习链接 |
|------|-----------|-----------|---------|
| **Token** | LLM 处理文本的最小单位，1 个中文字 ≈ 2 个 token | 决定 API 费用，是培训核心话题 | [OpenAI Tokenizer](https://platform.openai.com/tokenizer) |
| **上下文窗口** | 模型一次能"记住"的 token 上限 | 超出窗口的内容会被遗忘 | [Context Window Guide](https://arxiv.org/abs/2310.04988) |
| **System Prompt** | 给模型的角色设定和约束 | CLAUDE.md 就是 Agent 的 System Prompt | [Anthropic Docs](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/system-prompts) |
| **Temperature** | 控制输出随机性（0=确定，1=创意） | 写代码设 0，写文案设 0.7 | 在 Claude/DeepSeek 界面自己调参试试 |
| **Prompt Caching** | 不变内容只计费一次 | 省 90% System Prompt 费用 | [Anthropic Caching](https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching) |

**验证**：能用一句话解释"为什么 Token 数量决定 API 费用"。

---

### 🔴 必修 2：Claude Code 安装和使用（15 分钟）

培训以 Claude Code 终端为主要演示工具。来之前装好并跑通 5 个命令。

```bash
# 1. 安装
npm install -g @anthropic-ai/claude-code

# 2. 配置 API Key
export ANTHROPIC_API_KEY=你的key

# 3. 优化设置（培训中会用到的配置）
export CLAUDE_CODE_ENABLE_THINKING=false
export CLAUDE_CODE_AUTO_COMPACT=true

# 4. 首次启动
claude "hello world"

# 5. 必须熟悉的 5 个命令：
/tokens      # 查看当前会话 Token 消耗（培训核心）
/compact     # 压缩上下文（省钱关键）
/context     # 查看上下文内容
/init        # 生成 CLAUDE.md 项目文件
/help        # 查看所有命令
```

**验证**：`/tokens` 能显示数字，知道每个数字是什么。

### 🟡 推荐 1：Prompt Engineering 基础框架（30 分钟）

> 培训 L3 会用到这些基础套路，但不会从零教。课前了解即可。

| 技巧 | 一句话 | 示例 |
|------|--------|------|
| Zero-Shot | 不给例子，直接提问 | "把这段代码迁移到 ArkTS" |
| Few-Shot | 给 2-3 个示例再提问 | "参考 LoginPage.ets 的写法，迁移 HomePage" |
| Chain-of-Thought | 要求"逐步思考" | "分析这个模块的依赖关系，逐步拆解" |
| Role Prompting | 设定角色 | "你是 Android 转鸿蒙的专家" |
| Structured Output | 要求特定格式输出 | "只输出代码，不要解释" |

**推荐阅读**：
- [Anthropic Prompt Engineering Guide](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview)（选读前 3 节）
- [OpenAI Prompt Engineering](https://platform.openai.com/docs/guides/prompt-engineering)（浏览即可）

**验证**：能用上述 5 种技巧之一写一个 Prompt。

## 三、预习自测

来上课前，确认以下问题都能回答：

1. ❓ 什么是 Token？为什么它和 API 费用直接相关？

3. ❓ `/compact` 命令做了什么？为什么要用它？
4. ❓ System Prompt 是什么？CLAUDE.md 和它什么关系？
5. ❓ "Role Prompting" 是什么意思？自己写一个例子。
6. ❓ Few-Shot 和 Zero-Shot 的区别是什么？

## 五、FAQ

**Q: 我以前完全没用过 AI 写代码，能来吗？**
A: 可以。但必须预习完这个文档。培训目标是把"用过 AI 的人"变成"高效用 AI 的人"，需要先会走。

**Q: Claude Code 装不上怎么办？**
A: 确保 Node.js ≥ 18。或到教室提前 15 分钟找讲师帮忙。

**Q: 必须 Claude Code 吗？我用 Cursor/Copilot 可以吗？**
A: 培训演示用 Claude Code，但概念（Token/上下文/Skills）跨工具通用。建议提前装好 Claude Code 跟着做。
