## 二、预习清单（按优先级）

### 🔴 必修（1.5 小时，来之前必须完成）

#### 1. LLM 核心概念速通（30 分钟）

培训中频繁出现的词，课前请自行搞懂：

| 概念 | 一句话解释 | 为什么重要 | 预习链接 |
|------|-----------|-----------|---------|
| **Token** | LLM 处理文本的最小单位，1 个中文字 ≈ 2 个 token | 决定 API 费用，是培训核心话题 | [Understanding Tokens](https://platform.openai.com/tokenizer) |
| **上下文窗口** | 模型一次能"记住"的 token 上限 | 超出上下文窗口的内容会被遗忘 | [Context Window Guide](https://arxiv.org/abs/2310.04988) |
| **Temperature** | 控制输出随机性（0=确定，1=创意） | 写代码设 0，写文案设 0.7 | 自己试：Claude/DeepSeek 界面改 temperature |
| **System Prompt** | 给模型的角色设定和约束 | 决定 AI 的行为模式 | [Anthropic System Prompts](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/system-prompts) |
| **Embedding** | 将文本转为高维向量 | RAG、搜索、记忆系统的基础 | 不做硬性要求，有概念即可 |
| **Fine-tuning** | 在基础模型上用特定数据继续训练 | 培训不讲这个，但要知道区别于 Prompt | 知道"调整权重"vs"调整输入"即可 |

**验证**：能用自己的话解释"为什么 Token 数量决定 API 费用"。

#### 2. Claude Code 安装 + 5 个命令（15 分钟）

培训中 80% 的演示在 Claude Code 终端。来之前装好并跑通。

```bash
# 1. 安装
npm install -g @anthropic-ai/claude-code

# 2. 配置 API Key
export ANTHROPIC_API_KEY=你的key

# 3. 关掉扩展思考（培训中会用到的设置）
export CLAUDE_CODE_ENABLE_THINKING=false
export CLAUDE_CODE_AUTO_COMPACT=true

# 4. 跑第一次
claude "hello world"

# 5. 熟悉这 5 个命令
/tokens    # 查看当前会话消耗了多少 Token
/compact   # 压缩上下文（培训核心技巧）
/context   # 查看上下文内容
/init      # 生成 CLAUDE.md 项目文件
/help      # 查看所有命令
```

**验证**：`/tokens` 能显示数字，知道每个数字代表什么。

#### 3. Git 基础（15 分钟）

培训中有小组协作环节，需要基本的 Git 操作。

```bash
git clone <url>            # 拉取仓库
git add <file>             # 暂存修改
git commit -m "..."        # 提交
git push                   # 推送
git pull                   # 拉取最新
```

**验证**：能把一个本地修改推送到你自己的 GitHub 仓库。

### 🟡 推荐（1 小时，来之前尽量完成）

#### 4. Attention 机制直观理解（20 分钟）

> 培训模块三会讲"从 Attention 理解 Prompt 为什么有效"。课前知道 Attention 是什么就够了。

不需要推导数学公式，建立直觉：

```
Attention 的本质是"相关性加权"

输入句子："把 LoginActivity 迁移到 ArkTS"
Attention 做的事情：
  "迁移"  ← 高度关注 "LoginActivity" 和 "ArkTS"
  "把"    ← 几乎不关注任何词（虚词权重低）

→ Prompt 中每个词都在竞争模型的注意力
→ 冗余词稀释注意力 → 结果变差
→ 精确词聚焦注意力 → 结果变好
```

**推荐资源**：
- [The Illustrated Transformer](https://jalammar.github.io/illustrated-transformer/)（只看 Attention 部分，10 分钟）
- [3Blue1Brown — Attention in Transformers](https://www.youtube.com/watch?v=eMlx5fFNoYc)（可视化讲解，15 分钟）

**验证**：能回答"为什么 `迁移 LoginActivity 到 ArkTS` 比 `帮我把这个文件迁移到鸿蒙` 效果好？"

#### 5. Prompt Engineering 基础框架（15 分钟）

培训会讲 CRISPE 框架，课前知道 Prompt Engineering 有哪些基础套路即可。

| 技巧 | 一句话 |
|------|--------|
| Zero-Shot | 不给例子，直接提问 |
| Few-Shot | 给 2-3 个示例，再提问 |
| Chain-of-Thought | 要求"Let's think step by step" |
| Role Prompting | 设定角色："你是 XXX 专家" |
| Structured Output | 要求 JSON/Markdown 格式输出 |

**推荐阅读**：
- [Anthropic Prompt Engineering Guide](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview)（选读前 3 节，15 分钟）
- [OpenAI Prompt Engineering](https://platform.openai.com/docs/guides/prompt-engineering)（浏览即可）

**验证**：能用上述 5 种技巧之一写一个 Prompt（内容不限）。

#### 6. AI Coding 工具全景（10 分钟）

培训主要用 Claude Code。知道市面上有哪些类似工具、各自定位。

| 工具 | 定位 | 培训中使用 |
|------|------|:--:|
| Claude Code | Anthropic 原生，生态成熟 | ✅ 主力 |
| OpenCode | 开源，支持多模型 | 成本对比演示 |
| Cursor | IDE 内嵌 AI | 不提 |
| GitHub Copilot | 微软，代码补全 | 不提 |
| Aider | 开源 CLI | 不提 |

**知道即可，不需要安装。**

培训模块二会讲到在 Claude Code 中如何利用 Prompt Caching 省 Token。

- [Anthropic Prompt Caching](https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching)（10 分钟）
- 核心：不变的内容（System Prompt + CLAUDE.md）只需付一次 Token 费用

| 模型 | 输入每 1M token | 输出每 1M token | 适合 |
|------|:---:|:---:|------|
| Claude Opus 4 | $15 | $75 | 复杂审查 |
| Claude Sonnet 4 | $3 | $15 | 日常主力 |

## 五、FAQ

**Q: 我以前完全没用过 AI 写代码，能来吗？**
A: 可以。只要预习完这个文档。培训的目标就是把"新手"变成"能用 AI 提效的人"。

**Q: Token 概念我看了但还是不太懂？**
A: 培训模块二会专门讲 Token 消耗和优化。预习只需要知道"Token = 计费单位"，培训时会更深入。

**Q: 必须用 Claude Code 吗？我用 Cursor/Copilot 可以吗？**
A: 培训演示用 Claude Code。其他工具概念相似但命令不同，建议提前装好 Claude Code。
