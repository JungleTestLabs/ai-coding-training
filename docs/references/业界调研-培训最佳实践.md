## 二、注意力周期与防疲劳

| 原则 | 说明 | V3 实现 |
|------|------|---------|
| **25 分钟注意力极限** | 成人持续注意力不超过 25-30 分钟 | 每 25-30min 切换形式 |
| **形式轮换** | 同一形式连续不超过 30min | 7 种形式（讲授/Live/游戏/投票/动手/竞赛/展示） |
| **番茄节奏** | 上午 2 段间茶歇，下午 2 段间茶歇 | 每天 2 次 15min 茶歇 |
| **认知冲突** | 对比冲击引发好奇心 | 展示"没优化 vs 优化后"的 Token 数据 |
| **身体活动** | 投票站起/坐、白板贴便签、小组换座 | 每天 3-4 次身体移动 |

## 四-A：高级专业人士培训方法论（🆕 V3.1 采用）

### 为什么不用游戏化？

高级专业人士：
- 反感被"打分""排名"（觉得是幼儿园做法）
- 看重同行认可甚于外部奖励
- 学习动力来自解决真实问题，而非赢积分
- 对"培训"的期待是深度交流和可复用资产

### 替代方案

| 方法 | 说明 | V3.1 应用 |
|------|------|-----------|
| **案例研讨** | 真实项目深度剖析 | L4 代码走读 + Workshop 2 迁移 |
| **方案对比** | 同一问题多种解法，分析 trade-off | Workshop 2 多策略 + Workshop 3 同行互看 |
| **圆桌对话** | 讲师抛话题，学员主导讨论 | L5 三个话题圆桌 |
| **同行评审** | 互相审查代码，专业交流 | Workshop 3 结尾 |
| **开放空间** | 自组织讨论，参与者设定议题 | Day 2 结尾 |
| **认知冲突** | 对比冲击引发好奇，推动主动思考 | 每模块设计"对比时刻" |
| **自主选择** | 给学员控制权（选难度/选议题） | Workshop 3 三档自选 |
| **问题驱动** | 每模块以真问题开头，不灌输 | L1"为什么有人用AI反而更慢？" |

## 五、Peak-End 法则应用

心理学研究：人们对体验的评价主要取决于**峰值时刻**和**结尾时刻**。

| 峰值设计 | V3 位置 |
|---------|---------|
| **Day 1 峰值** | Workshop 2 迁移竞赛（45min 高强度 + 对比讲师答案的认知冲击） |
| **Day 2 峰值** | Workshop 3 端到端项目（75min 完整创造 + 展示成果的成就感） |
| **结尾** | 颁奖+合影（正向情绪收尾） |

## 七、新增参考：成人学习理论

| 理论 | 要点 | V3 应用 |
|------|------|---------|
| **Knowles 成人学习 6 原则** | 需要知道"为什么学"、自治、经验驱动、任务导向 | 每模块先回答"WHY"，用学员真实痛点驱动 |
| **Kolb 体验学习圈** | 体验→反思→概念化→实验 | Live Demo(体验)→拆解(反思)→讲授(概念)→Workshop(实验) |
| **ARCS 动机模型** | Attention→Relevance→Confidence→Satisfaction | 悬念(注意)→真实项目(相关)→阶梯任务(信心)→展示(满足) |
| **VARK 学习风格** | Visual/Auditory/Reading/Kinesthetic | 幻灯片(视觉)+讲授(听觉)+代码走读(阅读)+动手(动觉) |

## 九、新增参考：NPS 提升技巧

| 技巧 | 说明 |
|------|------|
| **超越预期** | 给学员一个"他没料到"的收获（如：一个可直接复用的Skill模板） |
| **解决真实痛点** | 让学员在培训中解决自己的项目问题，而非 toy example |
| **社交连接** | 午餐自由交流、小组竞赛建立同伴关系 |
| **可带走资产** | 6 项交付物确保培训结束后有实际可用的东西 |
| **WOW 时刻** | L3 Live Coding 从"一句话需求"到"编译通过的代码"仅需 75min |
| **结尾温暖** | 颁奖+合影+award ceremony 创造正向情绪 |
