# 业界调研 — AI Coding 培训最佳实践参考

> 2026-05-11 整理。为课程设计提供外部参考，标注可借鉴性。

## 一、AI Coding 工作坊设计原则

| 原则 | 说明 | 可借鉴性 | 本课程体现 |
|------|------|:--:|-----------|
| **70/20/10 时间分配** | 70% 实操 + 20% 讨论 + 10% 讲授，动手优先 | 高 | 实操(全流程推演+L4实战) 50% + 讨论(游戏/辩论) 30% + 讲授 20% |
| **阶梯式任务** | 补全→修改→构建→设计，逐级递进 | 高 | L1 补全+L2 理解→L3 构建→L4 设计 |
| **真实项目驱动** | 用学员自己的项目，非 toy example | 高 | L4 安卓转鸿蒙真实客户项目 |
| **配对编程模式** | 学员指导 AI，而非被 AI 替代 | 中 | L3 互动中体现：学员写 Prompt，AI 执行 |
| **产出导向** | 培训结束 = Prompt 库 + 规范文档 | 高 | 6 项交付物：Skill模板/CLAUDE.md/Prompt集 |

## 三、AI Native 开发最新趋势

| 趋势 | 说明 | 可借鉴性 |
|------|------|:--:|
| **Spec-as-Code** | 规格即代码，代码是 Spec 的编译产物 | ✅ 已融入 L3 |
| **AI First 架构** | 设计架构时先考虑 AI 可理解性 | 中 |
| **TDD → SDD** | 从测试驱动到规格驱动的范式转换 | ✅ 课程核心 |
| **三级 Spec 体系** | 产品 Spec → 技术 Spec → AI Spec | 中 |
| **Given-When-Then 双向生成** | Spec 可生成测试，测试也可反向生成 Spec | 中 |

## 四、Prompt Engineering 工作坊设计

| 要点 | 说明 | 可借鉴性 |
|------|------|:--:|
| **结构化框架** | Role→Context→Task→Constraints→Output Format→Example | 高（训前预习用） |
| **迭代实验方法论** | 观察输出→调整 Prompt→再试，形成闭环 | ✅ 融入训前 |
| **分领域模板** | 代码审查/测试生成/重构/文档各有专用 Prompt | ✅ 融入 L6 交付物 |
| **编辑原提示词代替追问** | 省 50% Token，结果更准 | ✅ 已在 V1 中保留 |

## 六、分层教学设计（供参考）

```
初级（第1月）    → 用 AI 写函数、改样式、生成测试
中级（第2-3月）  → 用 AI 设计模块、迁移代码、生成文档
高级（第4月+）   → 用 AI 理解架构、设计系统、制定规范
```

本课程定位：初级→中级的桥梁。学员来之前有基础使用经验，走的时候能进行模块级操作。
