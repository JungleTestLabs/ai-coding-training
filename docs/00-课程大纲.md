# AI Coding 实战培训 — 两天课程大纲（V3）

> 基于 specs/update2.md。从 1 天升级到 2 天。
> 上午：讲授+演示（每天 2 个模块）。下午：实战+展示（每天 ~3.5h 动手）。
> 面向终端、鸿蒙、计算等重点项目的内部开发者。

## 二、防疲劳与互动设计（核心方法论）

### 2.1 注意力节奏

```
每 25-30 分钟切换一次形式，打破疲劳曲线：

讲授(20-25min) → Live Demo(10-15min) → 提问互动(5min) → 动手(10min)
     ↑                                              |
     └──────────── 循环 ─────────────────────────────┘
```

**禁止**：连续讲授超过 30 分钟无互动。

### 2.2 七种活动形式轮换

| 形式 | 使用场景 | 时长 | 频率 |
|------|---------|:--:|:--:|
| 📖 结构化讲授 | 核心概念、框架、方法论 | 20-25min | 每天 3-4 次 |
| 🎬 Live Coding | 真实操作演示 | 15-20min | 每天 2-3 次 |
| 🎮 互动游戏 | 决策游戏、Prompt 对决 | 20-30min | 每天 1 次 |
| 🤔 提问快答 | Kahoot/举牌投票 | 5-10min | 每模块 1-2 次 |
| 💻 动手练习 | 终端上跟着做 | 15-30min | 每天 3-4 次 |
| 🏆 小组竞赛 | 限时完成任务 | 30-45min | 每天 1 次 |
| 🎤 展示分享 | 小组展示成果 | 10-15min/组 | 每天结尾 |

### 2.3 游戏化元素

| 元素 | 体现 |
|------|------|
| **小组积分制** | 每组初始 100 分，完成任务加分，回答问题加分 |
| **即时反馈** | Kahoot 投票实时显示结果，代码审查正确率排行榜 |
| **实物奖励** | 积分最高组获得奖品（技术书籍/周边） |
| **成就徽章** | "Token杀手"、"Prompt大师"、"Bug猎人" |
| **展示驱动** | 每组结尾必须展示成果，同伴压力转化为动力 |

### 2.4 防走神技巧

| 技巧 | 时机 |
|------|------|
| **神秘悬念** | 每模块开头扔一个"奇怪现象"("为什么同样的prompt，Token差了10倍？") |
| **鲜活案例** | 用学员项目中的真实代码，不用 toy example |
| **对比冲击** | 展示"没优化 vs 优化后"的极致对比 |
| **身体移动** | 投票站起/坐、白板贴便签、小组换座位 |
| **随机点名** | 提问时随机抽人，而非等自愿 |
| **倒计时压力** | 限时任务用大屏倒计时，制造紧迫感 |

## 三、各模块详细设计（V3）

---

### Day 1 上午：基础 + 架构

---

#### L1: AI Native 开发范式 (75min) | 09:30-10:45

**形式配比**：讲授 30min + Live Demo 20min + 互动 15min + 动手 10min

| 时间 | 形式 | 内容 |
|:---:|:--:|------|
| 09:30 | 📖 讲授 | **神秘悬念**："为什么有人用 AI 写代码反而更慢？" + 三层范式演进 |
| 09:55 | 🎬 Demo | 现场演示：同一需求，传统方式 vs AI Native 方式的效率对比 |
| 10:15 | 💻 动手 | 学员在自己的终端用 AI 实现一个小功能（限时 10min） |
| 10:25 | 🤔 互动 | 举手投票："你觉得 AI 最不可信的地方是？" → 引出三角色模型 |
| 10:35 | 📖 讲授 | SDD 融合 + 实战数据冲击（12 模块迁移的 Token 数据） |
| 10:45 | | ☕ 休息 |

**关键教学点**：（与 V2 相同，但节奏更紧凑，10:25 后不停顿直接切到数据冲击）

**神秘悬念**："为什么有人用 AI 写代码反而更慢了？"
→ 答案：因为还在用"传统开发习惯"——粘贴全文、不拆任务、不写 Spec。
→ 因此需要新的范式（AI Native）。

---

#### L2: Agent/Claude架构深度解读 (60min) | 11:00-12:00

**形式配比**：讲授 20min + Live Demo 25min + 动手 15min

| 时间 | 形式 | 内容 |
|:---:|:--:|------|
| 11:00 | 📖 讲授 | Agent 四层架构（上下文/Skills/记忆/驾驭）+ Token 消耗全景 |
| 11:20 | 🎬 Demo | **Live Coding**：现场 `/tokens` `/compact` `/context` 真实对比 |
| 11:35 | 💻 动手 | 学员在自己的终端跑 `claude` 进入项目，执行 `/tokens` 查消耗 |
| 11:45 | 🎬 Demo | Skills 机制演示：一个 Skill = Markdown → 复制即用 |
| 11:55 | 🤔 快问 | "`/compact` 能省多少 Token？" → 举手选择：30%/50%/70%/90% |

**关键变化**：把之前的"Token 震撼教育"从静态展示改为学员自己动手查 + 现场对比。

### Day 1 下午：实战入门

---

#### ☕ 午餐 (120min) | 12:00-14:00

**自由交流**：午间不强制安排，但鼓励跨项目交流。屏幕循环播放 Claude Code 操作录屏。

---

#### Workshop 1: 环境搭建 + 练手 (60min) | 14:30-15:30

**形式**：纯动手。讲师巡场一对一指导。

| 时间 | 形式 | 内容 |
|:---:|:--:|------|
| 14:30 | 💻 动手 | Claude Code 安装检查 + API Key 配置 + 环境变量设置 |
| 14:45 | 🎬 Demo | 讲师演示：创建一个 CLAUDE.md + 写一个 Skill |
| 15:00 | 💻 动手 | 学员任务：为 opencalc-harmonyos 写 CLAUDE.md + 用 AI 加一个功能 |
| 15:20 | 🎤 快展 | 2 组自愿展示：展示他们写的 CLAUDE.md |

**产出**：每位学员有自己的 CLAUDE.md + 第一次用 AI 完成代码修改。

---

#### ☕ 休息 (15min)

---

#### Workshop 2: 安卓转鸿蒙迁移实操 (75min) | 15:45-17:00

**形式**：限时挑战 + 小组竞赛。

| 时间 | 形式 | 内容 |
|:---:|:--:|------|
| 15:45 | 📖 讲授 | 迁移铁律速讲（5min）：Spec-First、分模块、4项验收 |
| 15:50 | 🏆 竞赛 | **Task**: 用 AI 将指定的 Android Activity 迁移到 ArkTS。限时 45min |
| | | 评分维度：编译通过(30分) / 功能完整(30分) / Token最少(20分) / 风格一致(20分) |
| 16:35 | 🎬 Demo | 讲师展示"标准答案" — 最优迁移方案 |
| 16:45 | 💻 动手 | 对照标准答案，学员修正自己的代码 |

**分组**：4-6 人一组，每组用不同的迁移策略（全自动/逐步/参考式），赛后对比效率。

---

#### 小组展示 + Day 1 回顾 (60min) | 17:00-18:00

| 时间 | 形式 | 内容 |
|:---:|:--:|------|
| 17:00 | 🎤 展示 | 每组 8min：展示迁移成果 + 遇到的坑 + Token 消耗 |
| 17:30 | 🏆 评分 | 讲师点评 + 公布 Day 1 积分排行 |
| 17:40 | 📖 回顾 | Day 1 核心知识点快闪（5 张幻灯片） |
| 17:50 | 🎮 悬念 | "明天：从一句话需求到完整功能，全流程 AI 推演" |

### Day 2 上午：全流程 + 案例

---

#### L3: 全流程AI Native推演 (75min) | 09:30-10:45

**形式配比**：Live Coding 45min + 讲授 20min + 互动 10min

| 时间 | 形式 | 内容 |
|:---:|:--:|------|
| 09:30 | 🎬 Live | **完整演示**："给 OpenCalc 加一个单位换算功能" — 从 Spec → Design → Code → Test。全程实时，不剪辑 |
| 10:15 | 📖 讲授 | 拆解刚才的演示：每一步在做什么？为什么这样做？ |
| 10:30 | 🎮 互动 | **Prompt 对决**：3 组用不同 Prompt 让 AI 生成同一个功能。投票"最强 Prompt" |
| 10:40 | 📖 讲授 | Spec-as-Code 理念 + 4 项验收标准 |

**关键变化**：把之前的"电影剧本五幕"压缩为"一段完整的 Live Coding"，让学员完整看到一个需求从零到交付的全过程。

#### ☕ 休息 (15min)

---

#### L4: 安卓转鸿蒙实战案例 + 代码走读 (60min) | 11:00-12:00

**形式配比**：案例研读 30min + 代码走读 30min

| 时间 | 形式 | 内容 |
|:---:|:--:|------|
| 11:00 | 📖 案例 | OpenCalc 迁移全景：12 模块、Token 从 $1,440 到 $3.50、6 条迁移铁律 |

| 11:45 | 📖 总结 | Skill 自包含、分层可组合、CI 验证、多平台适配 |

#### ☕ 午餐 (120min) | 12:00-14:00

---

#### L5: 工程最佳实践（游戏化）(60min) | 14:30-15:30

| 时间 | 形式 | 内容 |
