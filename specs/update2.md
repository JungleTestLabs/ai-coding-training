# AI Native 实战训战营 — 两天版课程规格

> 2026-05-11 | 基于 update1.md + 调研最佳实践
> 从 1 天 → 2 天。面向终端、鸿蒙、计算等内部重点项目的开发者。

## 核心变化

| 维度 | V2（1天版） | V3（2天版） |
|------|-----------|-----------|
| 时长 | 6h + 2h可选 | 2天，每天约 5.5h |
| 结构 | 连续讲授 | 上午讲授+演示 / 下午实战+展示 |
| 讲授比例 | ~60% | ~35%（更多实战） |
| 实战时间 | 0（可选2h） | ~5.5h（每天下午） |
| 防疲劳机制 | 无 | 番茄节奏 + 形式轮换 + 游戏化 |
| 目标学员 | 有 AI Coding 经验 | 终端/鸿蒙/计算等重点项目开发者 |

## 两天时间线总览

```
Day 1: 基础 + 实战入门
  上午 (9:30-12:00)    讲+演    L1 AI Native 范式 + L2 Agent/Claude架构
  下午 (14:30-18:00)   实战     环境搭建 + 安卓转鸿蒙动手 + 小组展示

Day 2: 全流程 + 真项目
  上午 (9:30-12:00)    讲+演    L3 全流程推演 + L4 实战案例
  下午 (14:30-18:00)   实战     L5 最佳实践游戏 + 端到端项目 + 展示评审 + L6 总结
```

## 防疲劳设计原则

| 原则 | 体现 |
|------|------|
| **25min 注意力周期** | 每 25-30min 切换形式（讲→演→练→问） |
| **形式轮换** | 讲授→Live Coding→互动游戏→动手→小组展示→点评，不连续同形式 |
| **番茄节奏** | 上午两段间茶歇；下午两段间茶歇 |
| **游戏化** | Day 2 下午 L5 用决策游戏代替纯讲授；小组竞赛制 |
| **展示驱动** | 每天下午结尾小组展示成果，制造 peak-end |
| **翻转课堂** | 核心概念训前自学，课堂时间留给理解和实践 |
