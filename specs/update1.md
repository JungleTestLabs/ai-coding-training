# Al Native 实战训战营:Al Coding 实战培训(突出特点、结合外部培训、减少重复工作)
巡站目标:面向内部开发者,基于安卓转鸿蒙实战经验,不讲Al基础,只讲能立刻用在工作中的深度内容。

## 课程定位
- 目标学员：有AI Coding开发经验，想用AI提升效率的开发者
- 学员痛点：AI代码不可信、不知道什么时候信AI、 Token烧钱
- 课程承诺：上完课第二天就能在项目里高效、高质量地使用AI
- 与市面课程的差异：不讲LLM/Transformer原理，讲最佳实践、复杂场景实战

## 核心差异化
- 不讲什么是LLM、Transformer基础、不讲AI历史和发展
- 基于Agent框架、SDD规范驱动开发流程和最佳实践
- 基于真实安卓转鸿蒙迁移项目的实战经验
- 每个模块有可操作输出物（Skill技能/项目最佳实践/Prompt模板等）

## 知识体系&课程大纲
- L1 AI Native 开发范式: SDD规范驱动 + AI的三层分工模型
- L2 Agent/Claude架构深度解读: Claude Code架构 + 源代码深度解密
- L3 全流程Al Native推演: 让学员看到AI在每个环节的具体用法
- L4 安卓转鸿蒙实战案例: 真是客户项目实战介绍和过程解读
- L5 工程最佳实践: Al审查清单+质量/安全/成本权衡
- L6 落地策略+总结: 团队推行+内源可复用资产+Q&A

## 全天时间线

### 训前
- LLM 核心概念速通: 培训中频繁出现的词，课前请自行搞懂
- Claude Code 安装和使用: 培训主要使用Claude Code 终端
- Prompt Engineering 基础框架: 课前知道 Prompt Engineering  有哪些基础套路

### 上午
- 0. 开场+ 破冰(09:00-09:15)
  - 领导讲话、开场破冰
- 1. AI Native 开发范式 (09:15 – 10:00)
  - 现在面临的痛点、传统 → AI增强 → AI Native
- 2. Agent/Claude架构深度解读 (10:00 – 11:00)
  - 上下文管理：拆解任务为可执行步骤
  - Skills技能：可复用的能力封装
  - 记忆存储 ：存储项目约定+ 会话历史
  - 驾驭工程：工具调用(文件读写/编译/搜索)
- 3. 全流程AI Native推演 (11:10 – 12:00)
  - 需求: Spec文档（AI辅助生成、人工确认）
  - 设计: 架构决策记录ADR（AI生成设计和任务）
  - 实现: 原子任务分解（AI执行）
  - 验证: 4项验收标准（编译/验证/修复/问题定位)

### 下午
- 4. 安卓转鸿蒙实战案例 (13:30 – 14:20)
  - Skill 自包含：独立工作，复制即用
  - 分层可组合：Pipeline 编排、Agent 调度 Skill
  - 知识可验证：CI 自动校验格式、防止知识腐化
  - 多平台适配：适配层处理平台差异
- 5. 工程最佳实践 (14:30 – 15:30)
  - 效率：追求用更少资源、更短时间达成目标
  - 质量：确保产出可靠、可维护、符合预期
  - 安全、可信：数据安全、符合规范
  - 成本：关注资源投入与长期效益的平衡
- 6. 落地策略 + 总结 (15:30 – 16:00)
  - 学员带着可操作的资产离开，第二天就能开始用
- 7. 培训结束 / 可选加时实操 (16:00 – 18:00)
  - 真实问题讨论、实战操作？
